1. ESPECIFICAÇÃO DOS CASOS DE USO
Caso de Uso 01
Identificador: UC-01
Nome: Autenticação de Usuário (Login)
Descrição: Permite que os usuários (alunos e professores) acessem o sistema 
PresenSee com suas credenciais para utilizar as funcionalidades de controle de 
presença.
Atores: Aluno, Professor.
Pré-condições: O usuário deve possuir um cadastro ativo e válido no sistema 
PresenSee.
Pós-condições: O usuário é autenticado, sua sessão é iniciada e ele é redirecionado 
para a tela inicial (Dashboard) correspondente ao seu perfil.
Fluxo Principal:
1. O usuário acessa a página inicial do PresenSee.
2. O sistema exibe o formulário de login solicitando E-mail e Senha.
3. O usuário insere seu E-mail e Senha.
4. O usuário clica no botão "Entrar".
5. O sistema valida as credenciais no banco de dados.
6. O sistema confirma a autenticação e redireciona o usuário para o Dashboard.
Fluxos Alternativos:
FA-01: Credenciais Inválidas: No passo 5, se o e-mail ou a senha estiverem incorretos, 
o sistema exibe a mensagem de erro "E-mail ou senha inválidos" e limpa o campo de 
senha.
FA-02: Campos em branco: No passo 4, se o usuário deixar algum campo vazio, o 
sistema bloqueia o envio e exibe um alerta indicando que "Todos os campos são 
obrigatórios".
Caso de Uso 02
Identificador: UC-02
Nome: Registro de Presença
Descrição: Permite que o aluno registre sua presença em uma aula ativa informando 
um código de chamada (ou lendo um QR Code) disponibilizado pelo professor.
Atores: Aluno.
Pré-condições: O aluno deve estar logado no sistema e o professor precisa ter aberto 
uma sessão de chamada ativa para a respectiva disciplina.
Pós-condições: A presença do aluno é registrada no banco de dados e computada no 
relatório da disciplina.
Fluxo Principal:
1. O aluno acessa a aba "Registrar Presença" no Dashboard.
2. O sistema exibe um campo de entrada para o "Código da Aula".
3. O aluno digita o código fornecido pelo professor.
4. O aluno clica no botão "Confirmar Presença".
5. O sistema verifica a validade e a vigência do código.
6. O sistema vincula o aluno à aula e exibe a mensagem de sucesso "Presença 
registrada com sucesso!".

Fluxos Alternativos:
FA-01: Código Inválido/Expirado: No passo 5, se o código digitado não existir ou a 
chamada já tiver sido encerrada, o sistema exibe a mensagem de erro "Código inválido 
ou aula encerrada".
FA-02: Presença já registrada: No passo 5, se o aluno já tiver computado presença para 
aquela sessão específica anteriormente, o sistema alerta "Atenção: Sua presença 
nesta aula já foi registrada".

Objetivo: Verificar se o sistema realiza login corretamente com dados válidos (Caminho

2. Preencher o e-mail.

principal (Dashboard).
